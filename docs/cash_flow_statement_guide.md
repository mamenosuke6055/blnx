# キャッシュ・フロー計算書 (C/F) — 家計簿アプリ実装ガイド

このドキュメントは、複式簿記の仕訳データからキャッシュ・フロー計算書 (Cash Flow Statement, C/F) を生成するための理論と実装方針をまとめたものです。`double_entry_bookkeeping_principles.md` を前提として読んでください。

---

## 1. なぜ家計簿に C/F が必要か

### 1.1 損益計算書 (P/L) では現金の流れが見えない

複式簿記を発生主義で運用すると、損益計算書 (P/L) は「いつ価値が動いたか」を示すが、「いつ現金が動いたか」は示さない。

例: クレジットカードで 30,000 円の家電を買った場合、

- P/L 上は購入日に「家電費 30,000 円」が計上される
- しかし実際に現金 (預金) が減るのは翌月の引落日

このギャップが家計のキャッシュ管理を難しくする。「黒字なのに口座残高が足りない」「赤字なのに残高は増えている」という現象は、P/L と現金収支の乖離から生まれる。

### 1.2 B/S と P/L の橋渡し

C/F は **期首の現金残高** と **期末の現金残高** の差を説明する書類。B/S のスナップショット間の現金変動を、P/L と整合する形で分解する。

```
期末現金残高 − 期首現金残高 = 営業 C/F + 投資 C/F + 財務 C/F
```

### 1.3 家計簿における C/F の意義

- 月次・年次で「実際にいくら現金が増減したか」を可視化
- 投資 (定期預金への移動、株式購入) と日常支出を分離して把握
- 借入金の返済負担と日常支出のバランスを評価
- 「黒字倒産」(家計版: 黒字なのに現金不足) の予防

## 2. キャッシュ (資金) の範囲を定義する

C/F を作る前に、まず **「キャッシュ」として扱う勘定の範囲** を定義する必要がある。これは企業会計でも家計簿でも最重要の前提。

### 2.1 一般的な定義: 現金及び現金同等物 (Cash and Cash Equivalents)

企業会計基準では「現金及び現金同等物」を以下の合計と定義する:

- **現金** — 手許現金、要求払預金 (普通預金、当座預金)
- **現金同等物** — 容易に換金可能で価値変動リスクが僅少な短期投資 (取得日から満期 3 ヶ月以内の定期預金など)

### 2.2 家計簿での推奨範囲

家計簿では実用性を優先して、次のように扱うのが分かりやすい:

**キャッシュに含める (Cash Accounts):**
- 現金
- 普通預金 (全口座)
- 電子マネー残高 (Suica、PayPay 残高、ICOCA など即時利用できるもの)

**キャッシュに含めない (Non-Cash Accounts):**
- 定期預金 (満期や解約手続きを要する)
- 投資信託、株式、暗号資産
- 前払金、未収金などの債権

**実装指針:**
- 勘定科目マスタに `is_cash: boolean` フィールドを持たせる
- ユーザー設定で個別の科目について `is_cash` を切替可能にしてもよい
- C/F 計算ロジックはこのフラグを参照して「キャッシュ勘定」を判定する

### 2.3 範囲を決めたら期中で変えない

期の途中でキャッシュの範囲を変更すると、期首・期末残高の比較が無意味になる。変更する場合は期末処理を挟むこと。

## 3. C/F の3区分

キャッシュ・フローは活動内容に応じて3つに区分する。

### 3.1 営業活動によるキャッシュ・フロー (Operating CF)

**日常的な収入・支出による現金の流れ。** 家計簿の文脈では、給与受取と生活費の支払いがほぼすべてここに入る。

**含まれる主な取引:**
- 給与・賞与の受取
- 副業収入の受取
- 食費、住居費、光熱費などの生活費支払い
- 税金、社会保険料の支払い
- クレジットカード引落 (生活費分)

### 3.2 投資活動によるキャッシュ・フロー (Investing CF)

**資産の取得・売却に伴う現金の流れ。** 将来の収益を生む資産への投下、またはその回収。

**含まれる主な取引:**
- 定期預金への預入・解約
- 投資信託、株式の購入・売却
- 不動産の購入・売却
- 高額な耐久財 (車、家電など) ※ 家計簿では費用処理することも多い
- 貸付金の実行・回収

### 3.3 財務活動によるキャッシュ・フロー (Financing CF)

**資金調達・返済に伴う現金の流れ。**

**含まれる主な取引:**
- 借入金の受取 (新規借入)
- 借入金の返済 (元本部分)
- 住宅ローンの借入・返済
- 奨学金の受取・返済

**注意:** 借入返済時の **利息部分は営業 C/F** に分類する (企業会計の慣行に従う)。これは利息が経常的な費用であるため。家計簿アプリの設計では、利息と元本を別エントリで仕訳しておくと自動分類しやすい。

### 3.4 区分の判定ルール (実装用)

仕訳内のキャッシュ勘定の反対側 (相手勘定) を見て区分を決める。

| 相手勘定の区分 | C/F 区分 |
|--------------|---------|
| 収益 (給与、利息収入など) | 営業 |
| 費用 (食費、光熱費など) | 営業 |
| 資産 (定期預金、投資信託、株式、不動産) | 投資 |
| 負債 (借入金 — 元本部分) | 財務 |
| 負債 (未払金 — クレジットカード) | 営業 (生活費の決済が多いため) |
| 純資産 | 財務 |

**実装指針:**
- 勘定科目マスタに `cf_category: 'operating' | 'investing' | 'financing'` フィールドを持たせて明示的に分類する
- 未払金 (クレカ) のように相手によって判定が変わる場合は、元の発生時の仕訳まで遡って判定するか、デフォルト分類を持たせる
- ユーザーが個別仕訳の C/F 区分を上書きできるようにしておくと安全

## 4. C/F の作成方法: 直接法と間接法

### 4.1 直接法 (Direct Method)

**現金収入・現金支出を主要な種類別に総額で表示する方法。**

```
[営業活動による C/F]
 給与収入                      3,000,000
 副業収入                        500,000
 食費の支払い                   △ 600,000
 住居費の支払い               △ 1,200,000
 光熱費の支払い                △ 180,000
 ...
 営業活動による C/F             1,020,000
```

**特徴:**
- 直感的でわかりやすい (家計簿ユーザー向き)
- キャッシュ勘定が関与する仕訳を集計するだけで作れる
- 家計簿アプリでは **これがメイン推奨**

### 4.2 間接法 (Indirect Method)

**純利益から出発して、現金収支のズレを調整して営業 C/F を算出する方法。**

```
[営業活動による C/F]
 当期純利益                      270,000
 (調整)
 未払金の増加                     30,000   ← まだ払っていない費用
 前払金の増加                   △ 10,000   ← 先に払った費用
 ...
 営業活動による C/F             1,020,000
```

**特徴:**
- 利益と現金収支のズレの原因が見える (B/S 項目の増減で表現)
- 企業の C/F は通常こちらで作られる (作成コストが低い)
- 家計簿では補助的な位置づけ

### 4.3 投資 C/F と財務 C/F は直接法・間接法で同じ

直接法・間接法の違いは営業 C/F の表示方法だけ。投資・財務 C/F は両方式で同じ形式 (総額表示) になる。

---

## 5. 直接法での C/F 算出アルゴリズム

家計簿アプリの主軸となるアルゴリズム。

### 5.1 基本ロジック

```
1. 期首・期末の日付を決める
2. キャッシュ勘定の集合を特定する (is_cash = true の勘定)
3. 期首時点のキャッシュ残高を計算
4. 期末時点のキャッシュ残高を計算
5. 期間内の全仕訳を取得
6. 各仕訳について、キャッシュ勘定を含むものだけを抽出
7. 相手勘定の cf_category で営業/投資/財務に振り分け
8. キャッシュ勘定側の動き (借方なら増加、貸方なら減少) を集計
9. 営業 C/F + 投資 C/F + 財務 C/F = 期末残高 − 期首残高 を検算
```

### 5.2 擬似コード

> 擬似コードは `side` 方式モデル（`double_entry_bookkeeping_principles.md` §3）で記述している。
> blnx の現行実装は符号付き金額規約のため、`side == 'debit'` ⇔ `value_num > 0` と読み替える。

```python
def generate_cash_flow_statement(period_start, period_end):
    cash_accounts = get_accounts(is_cash=True)

    opening_balance = sum(account.balance_at(period_start) for account in cash_accounts)
    closing_balance = sum(account.balance_at(period_end) for account in cash_accounts)

    operating_cf = 0
    investing_cf = 0
    financing_cf = 0

    transactions = get_transactions(date_range=(period_start, period_end))

    for tx in transactions:
        cash_entries = [e for e in tx.entries if e.account in cash_accounts]
        non_cash_entries = [e for e in tx.entries if e.account not in cash_accounts]

        if not cash_entries:
            continue  # キャッシュが動かない仕訳は C/F に影響しない

        for cash_entry in cash_entries:
            # キャッシュ勘定の増減 (定位置は借方なので、借方ならプラス、貸方ならマイナス)
            cash_delta = cash_entry.amount if cash_entry.side == 'debit' else -cash_entry.amount

            # 相手勘定の cf_category を取得 (複合仕訳の場合は按分が必要)
            for counter_entry in non_cash_entries:
                share = counter_entry.amount / sum(e.amount for e in non_cash_entries)
                allocated_delta = cash_delta * share
                category = counter_entry.account.cf_category

                if category == 'operating':
                    operating_cf += allocated_delta
                elif category == 'investing':
                    investing_cf += allocated_delta
                elif category == 'financing':
                    financing_cf += allocated_delta

    # 検算: 必ず一致するはず
    assert operating_cf + investing_cf + financing_cf == closing_balance - opening_balance

    return CashFlowStatement(
        period_start, period_end,
        opening_balance, closing_balance,
        operating_cf, investing_cf, financing_cf
    )
```

### 5.3 注意点

**複合仕訳の按分:**
1つの仕訳に複数の相手勘定が含まれる場合 (例: クレカ引落で複数の費用科目)、キャッシュの動きを各相手勘定の金額比で按分する必要がある。実装上ここが最もミスしやすい。

**キャッシュ間の振替:**
普通預金から普通預金 (別口座) への振替は、両方ともキャッシュ勘定なので C/F には現れない。これは正しい挙動。ただし、普通預金から定期預金への振替は「定期預金 = 投資 C/F」として現れる。

**符号の扱い:**
出力時には支出を `△` (三角) や赤字、括弧で表示するのが慣例。内部表現は符号付き数値で持ち、表示時に変換する。

## 6. 間接法での営業 C/F 算出

### 6.1 ロジック

```
当期純利益 (P/L から)
  + 営業活動に関連する負債の増加 (例: 未払金の増加)
  − 営業活動に関連する負債の減少
  − 営業活動に関連する資産の増加 (例: 前払金の増加)
  + 営業活動に関連する資産の減少
  − 投資 C/F・財務 C/F に分類される損益項目 (家計簿では通常なし)
= 営業活動による C/F
```

### 6.2 家計簿での簡易版

家計簿では複雑な調整項目は少ない。主に以下を考慮すればよい:

- クレジットカード未払金の期首・期末差額
- 前払金の期首・期末差額
- 未収収益、未払費用の期首・期末差額

### 6.3 直接法との整合性チェック

直接法と間接法で算出した営業 C/F は **必ず一致する**。両方を計算してテストで突き合わせると、データの整合性確認に強力。

## 8. 実装時のチェックリスト

### データモデル
- [ ] 勘定科目に `is_cash: boolean` フィールドがある
- [ ] 勘定科目に `cf_category: 'operating' | 'investing' | 'financing' | null` フィールドがある
- [ ] null は「C/F 区分の判定対象外」(キャッシュ勘定自体など) を意味する

### 集計ロジック
- [ ] キャッシュ勘定を含む仕訳のみを抽出している
- [ ] キャッシュ間の振替 (両側ともキャッシュ) は C/F に計上されない
- [ ] 複合仕訳の按分が正しく行われている
- [ ] 借方・貸方による符号判定が正しい

### 検算
- [ ] 営業 C/F + 投資 C/F + 財務 C/F == 期末キャッシュ残高 − 期首キャッシュ残高
- [ ] 直接法と間接法で算出した営業 C/F が一致する
- [ ] 期首残高 == 前期の期末残高

### 表示
- [ ] 支出を `△` または括弧で表示している
- [ ] 区分ごとの小計が表示されている
- [ ] 期首・期末残高の差額と全体合計が一致して見える形になっている
- [ ] 通貨フォーマット (カンマ区切り) が適用されている

### エッジケース
- [ ] 仕訳が1件もない期間でも空の C/F が生成される
- [ ] キャッシュ勘定間のみの振替で C/F が空になることをテストしている
- [ ] cf_category が未設定の勘定が現れた場合、警告またはデフォルト分類で処理する
- [ ] 期中にキャッシュ勘定のフラグが変わった場合の挙動を定義している (推奨: 期末まで変更不可)

## 9. P/L・B/S・C/F の関係 (整合性)

3つの財務諸表は連動している。実装した家計簿アプリで以下が常に成立することをテストで担保する。

### 9.1 B/S 間の関係

```
期末 B/S の純資産 − 期首 B/S の純資産 = 当期純利益 (− 引出 + 元入)
```

家計簿で「引出・元入」がない場合は単純に純利益分だけ純資産が増える。

### 9.2 P/L と C/F の関係

```
当期純利益 (P/L) ≠ 営業 C/F
```

両者が異なる原因 = 発生主義と現金主義のズレ = 間接法の調整項目。家計簿で発生主義を採用していると、必ずズレが発生する。これは異常ではない。

### 9.3 B/S と C/F の関係

```
期末 B/S のキャッシュ合計 − 期首 B/S のキャッシュ合計 = C/F の現金増減
```

これが一致しなければ、どこかにバグがある。**統合テストで必ずチェックすること。**

## 10. 用語集 (Glossary)

| 用語 | 英語 | 意味 |
|------|------|------|
| キャッシュ・フロー計算書 | Cash Flow Statement (C/F) | 一定期間の現金収支を3区分で示す財務諸表 |
| 現金及び現金同等物 | Cash and Cash Equivalents | C/F が対象とする「キャッシュ」の範囲 |
| 営業活動 | Operating Activities | 日常的な収入・支出 |
| 投資活動 | Investing Activities | 資産の取得・売却 |
| 財務活動 | Financing Activities | 資金調達・返済 |
| 直接法 | Direct Method | 現金収支を総額で表示する方法 |
| 間接法 | Indirect Method | 純利益から調整して営業 C/F を算出する方法 |
| フリー・キャッシュ・フロー | Free Cash Flow (FCF) | 営業 C/F − 投資 C/F (家計簿でも応用可) |

## 11. 補足: 家計簿ならではの応用指標

C/F のデータから、家計簿アプリで提供すると価値の高い派生指標。

### 11.1 フリーキャッシュフロー (家計版)

```
FCF = 営業 C/F − (投資 C/F のうち資産形成目的のもの)
```

「日常生活を維持しつつ自由に使える現金」の目安。マイナスなら借入や貯金取崩しに依存している状態。

### 11.2 貯蓄率

```
貯蓄率 = (営業 C/F + 投資 C/F のうち資産形成分) / 収入合計
```

純粋な貯蓄能力を示す。

### 11.3 ローン返済負担率

```
返済負担率 = |財務 C/F の返済額| / 営業 C/F の収入合計
```

借入返済が収入に対してどの程度の負担になっているか。

**このドキュメントを参照する際の注意:**

C/F は P/L と B/S が正しく作られていることが大前提。複式簿記の貸借が崩れていると C/F も崩れる。実装の優先順位は **複式簿記の整合性 → B/S・P/L → C/F** の順で固めること。§9 の整合性チェックは統合テストとして必ず組み込むことを強く推奨する。
