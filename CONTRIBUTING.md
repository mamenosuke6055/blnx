# Contributing to blnx

blnx は日本の金融機関 CSV を複式簿記 SQLite に取り込む個人ファイナンスのデータ配管です。
いちばん歓迎している貢献は **対応金融機関を増やす importer の追加** と
**金融機関側のフォーマット変更への追従** です。

## 開発環境

```bash
uv sync          # 依存のインストール（dev グループ含む）
uv run pytest    # テスト実行
```

## importer を追加する

1. `py/importers/detect_source.py` に判定パターンを追加する
   - ファイル名で判定できるならファイル名パターン（小文字でマッチされます）
   - できないなら先頭 30 行のヘッダ語によるコンテンツ判定
2. `py/importers/import_<source>.py` を作る
   - 既存の `import_rakutenbank.py`（単純な表形式）や `import_resona_bank.py`
     （全銀協系の固定列）が参考になります
   - 取引の同一性は FITID（`SHA256:` プレフィックスのハッシュ）で担保し、再取込で重複しないこと
3. `tests/test_import_<source>.py` を書く（後述の PII 規律を必ず守ってください）
4. 対応金融機関一覧（README）に 1 行追加する

## PII 規律（最重要）

**実データをそのままコード・テスト・issue・PR に貼らないでください。**

- テストのフィクスチャは必ず**架空の値**で合成する: 氏名は「ヤマダ タロウ」等の
  サンプル名義、店舗・組織名は「サンプル○○」、口座番号様の数列は架空の並び
- 実 CSV の構造を示したいときは、**ヘッダ行＋架空明細行**に作り替える
  （`examples/inbox/` のサンプルが見本です）
- 自分の名義・振込元など個人固有の分類キーワードはコードに書かず、
  `config/classify_local.json`（バージョン管理外）に置く設計です
  （`config/classify_local.sample.json` 参照）
- 金融機関の**書式**（列名・日付形式・文字コード）は公開情報として共有歓迎です。
  あなたの**取引内容**は共有しないでください

## バグ報告

- 取込が失敗する場合: 金融機関名・CSV の取得画面・ヘッダ行・エラー出力を添えてください
  （明細行は架空化するか伏せてください）
- 仕訳が不正になる場合: 期待した勘定と実際の勘定、再現できる架空の明細例を添えてください

## スコープについて

blnx の責務は**測定**（取込・整合・スナップショット）までです。予測・最適化・助言の
類は本リポジトリのスコープ外で、issue を立てていただいても原則 close します。
